## ðŸ“ Real-World DTO Examples & Use Cases

---

### 1. **Create User (with Validation)**

```ts
// create-user.dto.ts
import { IsEmail, IsString, IsOptional, IsInt, Min } from "class-validator";

export class CreateUserDto {
  @IsString()
  name: string;

  @IsEmail()
  email: string;

  @IsOptional()
  @IsInt()
  @Min(0)
  age?: number;
}
```

```ts
// user.controller.ts
@Post()
create(@Body() dto: CreateUserDto) {
  // auto-validates dto
  return this.userService.create(dto);
}
```

âœ… `@Body()` will **automatically validate** the incoming body based on this DTO if `ValidationPipe` is enabled globally.

---

### 2. **Get Users With Query Parameters**

```ts
// get-users.dto.ts
import { IsOptional, IsInt, IsString, Min } from "class-validator";

export class GetUsersDto {
  @IsOptional()
  @IsInt()
  @Min(1)
  page?: number;

  @IsOptional()
  @IsInt()
  @Min(1)
  limit?: number;

  @IsOptional()
  @IsString()
  search?: string;
}
```

```ts
@Get()

  return this.userService.findAll(query);
}
```

âœ… Cleanly maps query string like `?page=1&limit=10&search=john` into a validated class instance.

### 4. **Nested DTOs**

```ts

  @IsString()
  street: string;
}

  @IsString()
  name: string;

  return this.customerService.create(dto);

```

âœ… Use `@ValidateNested()` and `@Type()` from `class-transformer` to validate nested objects properly.

âœ… Keeps your response consistent and separate from internal models/entities.

## ðŸ” Best Practices

| Tip                                           | Why                                                     |
| --------------------------------------------- | ------------------------------------------------------- |
| âœ… Separate `CreateDto` and `UpdateDto`       | Avoid accidentally requiring fields for partial updates |

| âœ… Use Swagger decorators (`@ApiProperty`)    | Auto-generate Swagger docs                              |

## ðŸ§ª Common Mistakes

| Mistake                                         | Fix                                       |
| ----------------------------------------------- | ----------------------------------------- |
